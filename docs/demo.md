# 演示脚本(3 分钟)

> 适合销售演示、用户介绍、上线答辩使用。

## 准备

1. 已按 README 启动 MongoDB + 后端 + 前端
2. 已 `npm run seed:demo` 灌入示例数据
3. 浏览器开 2 个标签页:
   - **标签 A**:学生视角(用 `student / 123456` 登录)
   - **标签 B**:教师视角(用 `teacher / 123456` 登录)

---

## 第 1 幕 · 教师已有课程(30 秒)

**操作:标签 B**

> 👨‍🏫 看,我已经准备好了一门 Python 入门课,有 7 个课时 7 个作业,学生可以直接看到。

- 进入 **教师后台** → 显示 **"小象的 Python 入门课"** 卡片(已发布,绿色徽章)
- 点进去,展示 7 个课时(print / 变量 / 数字 / 输入 / 条件 / 循环 / 列表)
- 展示 7 个作业,有 **"自动判分"** 徽章的表示设了预期输出

---

## 第 2 幕 · 学生写代码并运行(60 秒)

**操作:标签 A**

> 🎒 同学们好,今天我们从第 1 课"你好,世界!"开始!

1. 进 **📚 课程** → 选 **"小象的 Python 入门课"**
2. 课时列表显示第 1 课有作业徽章
3. 滚到 **"✏️ 开始挑战:说出你的第一句话"**
4. 进入编辑器,默认有注释提示
5. 在编辑器写:
   ```python
   print("你好,Python!")
   ```
6. 点 **"▶️ 运行"**
7. 第一次会等 2-3 秒(Pyodide 加载),之后秒出
8. **🖥️ 运行结果** 框显示 `你好,Python!` ✅
9. 顶部出现绿色卡片:"🎉 运行结果与预期一致,你可以提交啦!"

---

## 第 3 幕 · 提交并自动判分(30 秒)

**操作:标签 A**

1. 点 **"📤 提交"** 按钮
2. 提示 **"🎉 已提交,系统判定为通过!"**
3. 顶部菜单进 **"👤 我的"** → 看到刚才的提交,徽章 **"✅ 通过"**

---

## 第 4 幕 · 教师批改(45 秒)

**操作:标签 B**

> 👨‍🏫 老师这边,马上就能看到学生的提交了。

1. 教师后台 → 课程编辑 → 找第 1 课的作业 → 点 **"📊 提交"**
2. 看到刚才学生的那条提交,状态 **"✅ 已通过"**
3. 点 **"📝 查看学生代码"** 展开 → 看到 `print("你好,Python!")`
4. 点 **"✏️ 批改"** → 输 **95 分** + 评语 **"写得不错!下次试试把名字改成自己的~"** → 提交
5. 状态变成 **"👨‍🏫 95 分"**

---

## 第 5 幕 · 学生看到分数(15 秒)

**操作:标签 A**

1. 切回学生标签 → 刷新 **"👤 我的"** 页面
2. 刚才那条提交,徽章从 **"✅ 通过"** 变成 **"👨‍🏫 95 分"**
3. 下方有 **"💬 老师评语:写得不错!下次试试把名字改成自己的~"**

---

## 收尾(15 秒)

> 🎉 这就是完整的"看课 → 写代码 → 运行 → 提交 → 批改"教学闭环!
> 从 5 年级开始,孩子可以独立完成,不需要家长在旁边帮忙装 Python。

---

## 备选演示场景

### 演示错误提示友好性

- 在编辑器写 `prnt("hi")`(拼写错)
- 运行 → **🖥️ 控制台** 红色报错
- 下方 **彩色卡片**:
  > 🤔 **这个名字你好像没告诉过 Python 哦~**
  > 💡 试试在前面加一行: m = '你的名字',把名字告诉 Python
- 演示把 NameError 翻译成小朋友能理解的话

### 演示自动判分边界

- 写 `print("你好,Python! ")`(多了个空格)
- 运行结果与预期输出 **不一致**(字符串精确比对)
- 提示"运行了,但输出和预期不一样哦,再检查一下?"
- 删掉空格,再运行 → 通过
