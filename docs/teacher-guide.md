# 教师使用指南

> 👨‍🏫 欢迎使用小象编程教师后台!几步就能建一门课~

## 1. 注册教师账号

1. 打开 **小象编程** 网站
2. 点 **"👉 立即注册"**
3. 选择 **"👨‍🏫 我是老师"**
4. 填写用户名、头像、昵称、密码
5. 填写 **教师邀请码**(联系学校管理员获取,默认是 `DEMO2026`)
6. 点 **"🚀 注册"**

## 2. 教师后台入口

登录后,顶部菜单会有 **"👨‍🏫 教师后台"**,点进去就能看到:

- 已创建的课程列表
- "➕ 新建课程" 按钮

## 3. 新建课程

1. 点 **"➕ 新建课程"**
2. 选个 **emoji 封面**(代表这门课的主题)
3. 填 **课程标题**(比如:"Python 入门第一课")
4. 填 **课程简介**(可选,一句话告诉同学们学什么)
5. 点 **"✅ 创建"**
6. 自动跳到课程编辑页

## 4. 编辑课程

### 4.1 基本信息

在课程编辑页顶部:

- **emoji 封面**:直接点输入框改
- **课程标题**:直接改
- **课程简介**:在下方文本框里改
- 点 **"💾 保存"** 保存

### 4.2 发布

- 点 **"🚀 发布"** 后,学生才看得到这门课
- 想撤回就再点 **"📥 取消发布"**

### 4.3 添加课时

1. 往下滚到 **"📖 课时"** 区域
2. 滚动到 **"➕ 添加新课时"** 卡片
3. 填:
   - **课时标题**(比如:"第 1 课:你好,世界!")
   - **课时内容**(支持 Markdown 语法,可以用 `# 标题`、`**加粗**`、代码块等)
   - **示例代码**(可选,会显示在课时页)
4. 点 **"✅ 添加课时"**

> 💡 课时内容推荐用 Markdown 写,语法很简单,点 [这里](https://markdown.com.cn/) 学一下。

### 4.4 编辑课时

每个课时卡片都有:

- 标题输入框(可改名)
- Markdown 内容(可改)
- 示例代码(可改)
- "💾 保存课时" 按钮
- "🗑️" 删除按钮

## 5. 添加作业

### 5.1 给某个课时加作业

在课时卡片上点 **"➕ 课时作业"** 按钮,会弹出作业编辑框,自动关联到这个课时。

### 5.2 不绑定课时的作业

在 **"🎯 作业"** 区域右上角点 **"➕ 不绑定课时的作业"**。

### 5.3 作业字段说明

| 字段 | 说明 | 必填 |
|------|------|------|
| 标题 | 作业名称(比如:"说出你的第一句话") | ✅ |
| 题目说明 | 告诉学生要做什么(支持多行) | 推荐 |
| 起始代码 | 学生打开时编辑器里的默认代码(可写 `# 提示:` 注释) | 推荐 |
| 预期输出 | 若填了,系统会自动判分(字符串精确比对,忽略首尾空白) | 否 |
| 满分 | 默认 100 分 | 是 |
| 绑定课时 | 可选,绑定后课时页有"开始挑战"按钮 | 否 |

### 5.4 自动判分 vs 人工批改

- **填了"预期输出"**:学生提交时,系统自动比对 → `通过` / `未通过`
- **没填"预期输出"**:状态是 `待批改`,老师要去后台手动打分

## 6. 批改作业

1. 在课程编辑页的 **"🎯 作业"** 列表,点作业的 **"📊 提交"** 按钮
2. 看到所有学生的提交列表(按时间倒序)
3. 每条提交可以:
   - 点 **"📝 查看学生代码"** 展开看
   - 点 **"🖥️ 查看运行输出"** 展开看
   - 点 **"✏️ 批改"** → 输入分数和评语 → 提交
4. 学生马上能在 **"👤 我的"** 看到分数

## 7. 最佳实践

- **课时从简到繁**:第一节课别太难,先 print 一个字
- **多 emoji**:学生在文字里看到 emoji 更愿意读
- **示例代码要可运行**:学生最喜欢"复制一下就能跑"的代码
- **预期输出别太长**:自动判分是字符串精确匹配,太长容易判错
- **每节课 1-2 个作业**:太多学生压力太大
- **批改写评语**:比起"80 分","80 分,for 循环少了 break,会死循环"对学生帮助大得多

## 8. 常见问题

**Q: 课程怎么删除?**
A: MVP 暂不支持删除,只能 "📥 取消发布" 让学生看不到。

**Q: 怎么生成新的邀请码?**
A: 修改 `backend/.env` 的 `TEACHER_INVITE_CODES` 字段(逗号分隔多个),然后重启后端。

**Q: 学生的代码乱了,我想重置?**
A: 改作业的 "起始代码" 字段 → 保存。下次学生打开就是新代码(但已经提交的不会变)。

**Q: 自动判分说"未通过",但我觉得学生写得对?**
A: 字符串完全一样才算通过。如果学生输出多了空格或少打了一个字,就不会通过。可以在"预期输出"前后留出容错空间(空字符串),或改成人工批改。
